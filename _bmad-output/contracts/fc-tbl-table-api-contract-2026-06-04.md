## Confirmed Public FC-TBL CLR Types

The current public FC-TBL surface under `Hexalith.FrontComposer.Shell.Components.DataGrid` is:

- `ColumnDescriptor`
- `ColumnVisibilityContext`
- `FcColumnFilterCell`
- `FcColumnPrioritizer`
- `FcExpandInRowDetail`
- `FcExpandedRowHiddenBanner`
- `FcFilterEmptyState`
- `FcFilterResetButton`
- `FcFilterSummary`
- `FcMaxItemsCapNotice`
- `FcNewItemIndicator`
- `FcProjectionGlobalSearch`
- `FcSlowQueryNotice`
- `FcStatusFilterChips`

The reflection baseline includes exported types, public constructors, declared public properties, declared public fields, and declared public non-special methods. It therefore covers component parameter properties such as `FcColumnFilterCell.ColumnHeader`, `FcColumnPrioritizer.MaxVisibleColumns`, `FcExpandInRowDetail.PanelId`, and `FcExpandInRowDetail.SuppressedAnnouncement`.

## Boundary Decision

The 12 Story 1.0 DataGrid components still exist as public `ComponentBase` types. `ColumnDescriptor` and `ColumnVisibilityContext` are also public because the prioritizer child-fragment contract exposes them. No component was internalized in this story.

The package validation mechanism in `Directory.Build.targets` is opt-in and package-version based, so it does not by itself pin the local Shell FC-TBL surface during normal test runs. Story 2.8 therefore adds the focused baseline and test. A future full Shell public API baseline can widen the same mechanism beyond the FC-TBL namespace without changing this contract's disposition.
