# Story

## Story Statement

**As a** business user,
**I want** a command palette that lets me fuzzy-search across all projections, commands, and recent views from anywhere in the application,
**So that** I can navigate and take action in 2 keystrokes instead of clicking through menus.

## Story Goal (one sentence)

Ship the framework-owned `FcCommandPalette` + `IShortcutService` so `Ctrl+K` anywhere in a Counter.Web-style adopter opens a `FluentDialog`-hosted `FluentSearch` that fuzzy-matches against registered projection names, command names, and recent-route history ‚Äî categorised into **Projections / Commands / Recent** with sub-100 ms scoring (NFR5), contextual-mode bonus for the current bounded context, `role="listbox"` keyboard navigation, screen-reader result-count announcements, and a `shortcuts` special query surfacing every registered binding ‚Äî while registering the three v1 shell shortcuts (`Ctrl+K` open-palette, `Ctrl+,` open-settings, `g h` go-home) through a duplicate-conflict-warning (`HFC2108` Information) service that retires the Story 3-3 D16 inline binding.

## Scope Statement

Story 3-4 is the **Epic 3 power-user navigation body** that combines two tightly-coupled concerns: the keyboard-shortcut registration surface (so the shell can own `Ctrl+K` without fighting adopter bindings) and the command-palette UX that `Ctrl+K` opens. It consumes Story 3-3's `IDialogService` wiring (dialogs are Fluent UI-owned, not Fluxor-tracked), Story 3-2's `IFrontComposerRegistry` manifests (the search index), Story 3-1's `IStorageService` + `IUserContextAccessor` fail-closed scope guard (for recent-route persistence), and Story 1-3's Fluxor per-concern feature pattern. It opens a clean seam for Story 3-5's `IBadgeCountService` ‚Äî badges gracefully degrade to "not shown" when the service is absent and activate automatically when Story 3-5 registers it.

**In scope:**

- `src/Hexalith.FrontComposer.Contracts/Shortcuts/IShortcutService.cs` ‚Äî public interface with `IDisposable Register(string binding, string descriptionKey, Func<Task> handler)`, `IReadOnlyList<ShortcutRegistration> GetRegistrations()`, `Task<bool> TryInvokeAsync(KeyboardEventArgs)`. Binding string grammar: lowercase `ctrl+key` / `shift+ctrl+key` / bare-chord `g h` with a single space separating the two keys. Normalisation via `ShortcutBinding.Normalize(string)` helper (exported as `public static`). `Register` returns an `IDisposable` ‚Äî disposing unregisters. Duplicate registration (same normalised binding) logs `HFC2108_ShortcutConflict` at Information severity with both call sites + descriptions; **last writer wins** (replace semantics) so adopter code can deliberately override a framework binding by registering after `AddHexalithFrontComposer`.
- `src/Hexalith.FrontComposer.Contracts/Shortcuts/ShortcutRegistration.cs` ‚Äî `public sealed record ShortcutRegistration(string Binding, string DescriptionKey, string NormalisedLabel)`. `NormalisedLabel` is pre-computed at registration time (e.g., `"Ctrl+K"` from the lowercase binding) so the palette's "shortcuts" reference view doesn't re-compute per render.
- `src/Hexalith.FrontComposer.Shell/Shortcuts/ShortcutService.cs` ‚Äî internal implementation. **Scoped lifetime** (per-circuit in Blazor Server, per-user in WASM ‚Äî mirrors Story 3-1 `IStorageService` scoping ADR-030). Backing store: `ConcurrentDictionary<string, ShortcutEntry>` keyed on normalised binding. `TryInvokeAsync(KeyboardEventArgs)` normalises the incoming event into the same binding string format and looks up the handler; if found, awaits it and returns `true`. Two-key chord state (`g h`) is tracked via a short-lived (1500 ms) "pending first key" field ‚Äî a non-matching second key within the window clears it.
- `src/Hexalith.FrontComposer.Shell/Shortcuts/FrontComposerShortcutRegistrar.cs` ‚Äî hosted-service-style component that invokes `IShortcutService.Register("ctrl+k", "PaletteShortcutDescription", () => Dispatcher.Dispatch(new PaletteOpenedAction(UlidFactory.NewUlid())))`, `Register("ctrl+,", "SettingsShortcutDescription", () => DialogService.ShowDialogAsync<FcSettingsDialog>(...))`, and `Register("g h", "HomeShortcutDescription", () => NavigationManager.NavigateTo("/"))` on circuit init. **D24 idempotency**: `RegisterShellDefaultsAsync()` opens with `if (_registered) return; _registered = true;` ‚Äî guards against repeated calls from bUnit / hot-reload / prerender-then-interactive boot paths that can otherwise re-fire the three registrations and spam `HFC2108` logs. **Retires Story 3-3 D16 inline `@onkeydown` binding on `.fc-shell-root`** ‚Äî the shell now routes all keyboard events through `TryInvokeAsync`, with an early-return on bare text input targets (see D3).
- `src/Hexalith.FrontComposer.Shell/Routing/CommandRouteBuilder.cs` ‚Äî new tiny file exposing `public static class CommandRouteBuilder { public static string KebabCase(string pascal); public static string BuildRoute(string boundedContext, string commandTypeName); }`. Used by `HandlePaletteResultActivated` to build canonical `/domain/{bc}/{cmd}` URLs (D21). Lives under `Routing/` rather than co-located with `ShortcutBinding` because kebab routing is a command-routing concern ‚Äî NOT a keyboard-shortcut concern.
- `src/Hexalith.FrontComposer.Shell/State/CommandPalette/FrontComposerCommandPaletteFeature.cs` + `FrontComposerCommandPaletteState.cs` + `CommandPaletteActions.cs` + `CommandPaletteReducers.cs` + `CommandPaletteEffects.cs` ‚Äî new per-concern Fluxor feature following the Story 1-3 / 3-1 / 3-2 / 3-3 pattern. State: `public record FrontComposerCommandPaletteState(bool IsOpen, string Query, ImmutableArray<PaletteResult> Results, ImmutableArray<string> RecentRouteUrls, int SelectedIndex, PaletteLoadState LoadState)`. Initial: `(false, "", Empty, Empty, 0, Idle)`. Actions: `PaletteOpenedAction(CorrelationId)`, `PaletteClosedAction(CorrelationId)`, `PaletteQueryChangedAction(CorrelationId, Query)`, `PaletteResultsComputedAction(Query, Results)` (effect-originated), `PaletteSelectionMovedAction(Delta)` (+1/‚àí1), `PaletteResultActivatedAction(SelectedIndex)`, `RecentRouteVisitedAction(Url)`, `PaletteHydratedAction(RecentRouteUrls)`. Reducers stay pure static ‚Äî scoring is NOT invoked in the reducer (the effect computes `Results` before dispatching `PaletteResultsComputedAction`).
- `src/Hexalith.FrontComposer.Shell/State/CommandPalette/PaletteResult.cs` ‚Äî `public sealed record PaletteResult(PaletteResultCategory Category, string DisplayLabel, string BoundedContext, string? RouteUrl, string? CommandTypeName, int Score, bool IsInCurrentContext)`. `Category` is an enum `{ Projection, Command, Recent, Shortcut }`. Serialised into the Fluxor action payload; NEVER persisted.
- `src/Hexalith.FrontComposer.Shell/State/CommandPalette/PaletteScorer.cs` ‚Äî `public static class` exposing `Score(string query, string candidate) : int` (returns `0` for no match, higher is better). Algorithm: case-insensitive. **Exact-prefix** match scores `100 + length-of-match * 2`. **Contains substring** scores `50 + length-of-match`. **Fuzzy subsequence** (query chars appear in order in candidate, not necessarily contiguous) scores `10 + matched-char-count ‚àí gap-count`. Non-match ‚Üí `0`. Plus ADR-043's contextual bonus: `+15` when `IsInCurrentContext == true` (applied by the effect, not the pure scorer). Pure function ‚Äî no state, no DI, no locale conversion (ToLowerInvariant only). Property-test-covered with FsCheck (Task 10.3a).
- `src/Hexalith.FrontComposer.Shell/State/CommandPalette/CommandPaletteEffects.cs` ‚Äî effects:
    - `HandlePaletteQueryChanged(PaletteQueryChangedAction, IDispatcher, CancellationToken)` ‚Äî debounces 150 ms via `Task.Delay(150, ct)` with a per-dispatch `CancellationTokenSource` so later keystrokes cancel earlier awaits. After the delay, enumerates `IFrontComposerRegistry.GetManifests()`, scores every projection + command candidate via `PaletteScorer.Score(query, candidate.Name)`, adds the `IsInCurrentContext` bonus when `NavigationState.CurrentBoundedContext == candidate.BoundedContext`, takes top-50 by score, interleaves with recent-route matches, and dispatches `PaletteResultsComputedAction(query, results)`. On the literal query `"shortcuts"` (case-insensitive exact match), bypasses scoring and returns all `IShortcutService.GetRegistrations()` wrapped as `PaletteResult` with `Category = Shortcut`.
    - `HandlePaletteOpened(PaletteOpenedAction, IDispatcher)` ‚Äî dispatches `PaletteResultsComputedAction("", <recent + top-projections>)` so the palette opens populated with Recent category + a default Projections preview (no-op when registry empty).
    - `HandlePaletteResultActivated(PaletteResultActivatedAction, IDispatcher, NavigationManager)` ‚Äî reads `state.Results[action.SelectedIndex]`, navigates via `NavigationManager.NavigateTo(result.RouteUrl)` for `Category == Projection | Recent | Shortcut` (shortcuts with a route, e.g., `g h` ‚Üí "/") OR opens the command form dialog for `Category == Command` (dispatches the generator-emitted `{Command}FormOpenedAction` or navigates to the generated `/domain/{command}` page ‚Äî whichever path Story 2-2 established per its D14 render-mode rules). Always dispatches `PaletteClosedAction` + `RecentRouteVisitedAction(newRoute)` after successful activation.
    - `HandleRecentRouteVisited(RecentRouteVisitedAction, IDispatcher)` ‚Äî prepends the new route to the Recent ring buffer (max 5, de-dup on `string == string`), persists the ring buffer to `IStorageService` under `StorageKeys.BuildKey(tenantId, userId, "palette-recent")` with the same fail-closed scope guard used by Story 3-1 / 3-2 / 3-3 (`HFC2105_StoragePersistenceSkipped`).
    - `HandleAppInitialized(AppInitializedAction, IDispatcher)` ‚Äî loads the persisted ring buffer and dispatches `PaletteHydratedAction(storedUrls)`. Hydrate is read-only (ADR-038 mirror) ‚Äî does NOT re-persist.
- `src/Hexalith.FrontComposer.Shell/Components/Layout/FcCommandPalette.razor` + `.razor.cs` + `.razor.css` ‚Äî dialog content component, implements `IDialogContentComponent`. Body: `FluentSearch` input bound two-way to `Query` (via local field ‚Äî dispatch on change); `FcPaletteResultList` rendering the categorised results; visually-hidden `aria-live="polite"` region for "N results" announcements (skip first render per WCAG). Auto-focuses the search input in `OnAfterRenderAsync(firstRender)` via `FluentSearch`'s `FocusAsync()` API. On `KeyDown` the component handles Escape ‚Üí `PaletteClosedAction` + `Dialog.CloseAsync()`; ArrowUp / ArrowDown ‚Üí `PaletteSelectionMovedAction(-1/+1)`; Enter ‚Üí `PaletteResultActivatedAction(SelectedIndex)`. `role="dialog"` and `aria-label="CommandPaletteTitle"` carried on the dialog root (set via `DialogParameters.Title` + `aria-label` on the content component's root element).
- `src/Hexalith.FrontComposer.Shell/Components/Layout/FcPaletteResultList.razor` + `.razor.css` ‚Äî pure-presentational listbox. Parameters: `[Parameter] ImmutableArray<PaletteResult> Results`, `[Parameter] int SelectedIndex`, `[Parameter] EventCallback<int> OnSelectionChanged`. Renders `<ul role="listbox" aria-activedescendant="@ResultElementId(SelectedIndex)">` with `<li role="option" id="@ResultElementId(i)" aria-selected="@(i == SelectedIndex)">` sub-rendered groups (Projections / Commands / Recent / Shortcut ‚Äî each with a `role="group"` heading). `IBadgeCountService` is resolved via `ServiceProvider.GetService<IBadgeCountService>()` in `OnInitialized` (NOT `[Inject]` ‚Äî see ADR-044): when the service is registered AND the result's `Category == Projection`, appends a `FluentBadge` with the badge count; absent service ‚Üí no badge, no placeholder (AC7).
- `src/Hexalith.FrontComposer.Shell/Components/Layout/FcPaletteTriggerButton.razor` + `.razor.cs` ‚Äî header icon (`Icons.Regular.Size20.Search`) that dispatches `PaletteOpenedAction` on click. Auto-populates `FrontComposerShell.HeaderEnd` **before** the Story 3-3 `FcSettingsButton` (both populated when `HeaderEnd is null`; order is: palette trigger then settings). Symmetric pattern to Story 3-3 D12 ‚Äî adopter-supplied non-null `HeaderEnd` wins and suppresses both auto-populated buttons.

The `HeaderEnd` auto-populate branch (Story 3-3 D12) grows to include `<FcPaletteTriggerButton />` ahead of `<FcSettingsButton />`.

**Out of scope (Known Gaps / downstream stories):**

- Build-time Roslyn analyzer that enforces `IShortcutService.Register` unique-binding at compile time (so adopter code can't accidentally ship a conflicting registration) ‚Üí **Story 9-4** (diagnostic governance bucket; companion to Story 3-3 G15 grep-tripwire replacement).
- `IBadgeCountService` implementation + registration ‚Äî palette consumes it via nullable injection (AC7) ‚Üí **Story 3-5** (FcHomeDirectory, Badge Counts & New Capability Discovery).
- Real-time badge-count refresh via SignalR hub subscription ‚Äî palette results re-render on `IBadgeCountService.CountChanged` ‚Äî **Story 3-5** ships the service with the subscription; 3-4's palette component ALREADY listens via `IObservable<BadgeCountChangedArgs>` subscription in `FcPaletteResultList.OnInitialized` guarded by nullable injection, so when 3-5 lands the updates flow without a palette change ‚Üí tracked as **G4** per L10 discipline.
- Per-adopter custom palette categories (e.g., adopter registers "Reports" as a 4th category) ‚Üí **v1.x** (adopter-extensibility is a broader Epic 6 concern).
- Command invocation directly from the palette (type "increment counter" ‚Üí press Enter ‚Üí confirmation dialog appears with an Amount of 1) ‚Üí **v1.x**. 3-4 activates the command's generated form page via navigation (`NavigationManager.NavigateTo(generatedFormRoute)`); in-palette inline invocation requires `FluentSearch` to host an embedded command form, which inflates the dialog scope beyond the epic AC.
- Fuzzy scoring internationalisation (Turkish-I, Unicode normalisation, locale collation) ‚Üí **v2**. v1 ships `ToLowerInvariant` + ordinal comparison ‚Äî sufficient for EN + FR bounded-context names which do not exercise the hard cases.
- Virtual scrolling of palette results when result count > 50 ‚Üí **v2**. 3-4 caps top-N at 50 which is rendered non-virtualized (well under Fluent UI's row-limit tolerance).
- `forced-colors` + RTL verification of the palette dialog ‚Üí **Story 10-2** / **v2** (mirrors Story 3-3 G8 / G9 deferrals).
- Playwright screenshot diff of the palette across light/dark themes √ó Compact/Comfortable/Roomy density ‚Üí **Story 10-2** (density parity specimen pipeline inherits the preview panel from 3-3).
- `aria-describedby` linking the search input to the result count live region (a finer accessibility wire) ‚Üí **v1.x** ‚Äî 3-4 ships `aria-live="polite"` on the count region which screen readers auto-announce without the explicit describedby link; add the link in a follow-up if an a11y audit flags it.
- Multi-tenant palette result filtering by tenant registry (different tenants see different bounded contexts) ‚Üí **v1.x** ‚Äî v1 surfaces whatever `IFrontComposerRegistry.GetManifests()` returns; tenant-scoped manifest filtering is Epic 7 scope.
- CSS-in-JS / scripted keyboard-shortcut display ("press ‚åò+K" on macOS) ‚Üí **v1.x**. 3-4 ships the fixed "Ctrl+" label family; platform-appropriate glyph substitution is a v1.x enhancement.
