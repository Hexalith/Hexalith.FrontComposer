# Dev Agent Cheat Sheet (Read First — 2 pages)

> Amelia-facing terse summary. Authoritative spec is the full document. Every line links to a section for detail.

**Goal:** Land the framework-owned **collapsible sidebar** with three-tier responsive behaviour that replaces the adopter-authored `FluentNav` stopgap from Story 3-1 D25. Deliver (1) `FrontComposerNavigation.razor` consuming `IFrontComposerRegistry.GetManifests()` as one `FluentNavCategory` per bounded context with one `FluentNavItem` per projection — commands excluded (D1); (2) a convention-based route `/{boundedContext-lowercase}/{projectionTypeName-kebab-case}` (D2); (3) `Shell/State/Navigation/` Fluxor feature (`FrontComposerNavigationState`, `NavigationActions`, `NavigationReducers`, `NavigationEffects`, `ViewportTier` enum) persisted per tenant/user with the Story 3-1 ADR-029 fail-closed pattern (D3, D12); (4) `FcLayoutBreakpointWatcher.razor` + `fc-layout-breakpoints.js` that dispatch `ViewportTierChangedAction` whenever the browser crosses 1366 / 1024 / 768 px (D5, D6); (5) `FcCollapsedNavRail.razor` — 48 px icon rail used **only at CompactDesktop (1024–1365 px)** to resolve the FluentNav-has-no-icon-only-mode gap (D13); (6) `FcHamburgerToggle.razor` wrapping `FluentLayoutHamburger` in the shell `HeaderStart` slot, visible whenever `viewport ≠ Desktop` OR user manually collapses at Desktop (D8, D9); (7) one NEW append-only `HeaderCenter` RenderFragment parameter on `FrontComposerShell` (breadcrumb slot — populated by Story 3-5) (D10); (8) auto-population of `FrontComposerShell.Navigation` when adopter leaves the slot null AND the registry has ≥1 manifest (D18); (9) Counter.Web `MainLayout.razor` rewired to its three-line form (the adopter `FluentNav` block introduced by Story 3-1 D25 is DELETED) (D17).

**Scope boundary:** 3-2 is the **Epic 3 navigation body** that follows 3-1's shell skeleton. Badge counts on nav items (ActionQueue projections) → **Story 3-5**. Density override at <1024 px (`ViewportTier` consumer) → **Story 3-3**. Command palette / Ctrl+K → **Story 3-4**. Home directory content → **Story 3-5**. DataGrid filter/sort/scroll state restoration across nav → **Story 3-6**. Projection role-hint metadata + friendly names on `DomainManifest` → **Epic 4** (projection rendering). Breadcrumb content (the new `HeaderCenter` slot) → **Story 3-5**. Drag-to-reorder / favourites → **v2**. RTL / `forced-colors` / `axe-core` automation → **Story 10-2**. 3-2 inherits — does not re-litigate — Story 3-1's zero-override invariant, `LocalStorageService` scoping (ADR-030), `IUserContextAccessor` fail-closed default (ADR-029), six semantic color slots (ADR-031), and the scoped CSS bundle filename contract (ADR-034).

**ADR-035 one-liner:** `FrontComposerNavigation` auto-registers as the default `FrontComposerShell.Navigation` content — adopters who leave the render fragment null get the framework sidebar for free when `IFrontComposerRegistry.GetManifests()` returns ≥ 1 manifest; adopters who supply their own fragment still win (opt-out path preserved). Rejected: always render framework sidebar (breaks Story 3-1 D25 escape-hatch discipline); require explicit `<FrontComposerNavigation />` call (adds one line of boilerplate to every adopter `MainLayout`); emit a build warning if the slot is empty and registry has manifests (Roslyn analyzer cost > ergonomic benefit).

**ADR-036 one-liner:** Three viewport tiers are derived via JS `matchMedia` (composed from three queries `(min-width: 1366px)`, `(min-width: 1024px)`, `(min-width: 768px)`) and reported to C# via `DotNetObjectReference.invokeMethodAsync("OnViewportTierChangedAsync", tierInt)`. Rejected: pure CSS container queries (Story 3-3 density override needs reactive Fluxor state — a CSS-only toggle can't trigger the density forcing effect); `FluentLayout.OnBreakpointEnter` alone (single breakpoint at 768 px; not enough granularity for the 3-tier matrix); `ResizeObserver` on the shell element (equivalent noise, loses the semantic "tier" abstraction).

**ADR-037 one-liner:** `ViewportTier` is derived-at-runtime from the current browser window — NEVER persisted to `LocalStorageService`. `NavigationState.CurrentViewport` is set by `ViewportTierChangedAction` reducer and omitted from the persistence blob serialized in Task 3.3. Rejected: persist viewport (bootstrap paradox — page loads serverside at "last viewport" before JS runs so UI renders in the wrong tier); persist the user's "preferred" tier (conflates preference with observation — density + sidebar preferences are the knob, tier is just the signal).
