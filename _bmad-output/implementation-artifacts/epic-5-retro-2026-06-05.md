### What Went Well

1. **Contract-first MCP delivery worked.** Every story recorded a v1 contract artifact before or
   alongside tests, making command-tool ordering, lifecycle shape, resource grammar, fail-closed
   behavior, and schema taxonomy reviewable.

2. **Identity hardening closed a real risk.** Story 5.1 removed GUID and `Activity.TraceId`
   fallbacks from MCP command identity. `MessageId`, `CommandId`, and `CorrelationId` now come from
   `IUlidFactory`, and rejected caller-supplied identity stops before allocation or dispatch.

3. **Hidden-equivalent security became consistent.** Story 5.4 fixed the hidden-tool name oracle and
   pinned parity across absent, tenant-hidden, policy-hidden, and auth-failed paths. Story 5.5 then
   preserved that precedence over schema diagnostics.

4. **Schema negotiation is placed before side effects.** Story 5.5 proved incompatible or stale
   schema decisions stop before command construction, ULID allocation, lifecycle tracking, dispatch,
   projection query, or render work.

5. **Resource boundaries are clearer than the epic shorthand.** Story 5.3 resolved projection URIs
   to the generated grammar `frontcomposer://<bounded-context>/projections/<projection-name>` and
   kept skill resources under the reserved `frontcomposer://skills/` prefix.

### Challenges And Growth Areas

1. **File List drift still recurred.** Story 5.1 needed review-side File List corrections, and Story
   5.5 had a critical review finding because four E2E/QA deliverables were missing from the File
   List while the reconciliation task was marked complete.

2. **Local exact test execution remains environment-sensitive.** The exact solution-level test lane
   repeatedly hit local socket restrictions before test execution. Fallback evidence is strong, but
   the project still needs a clearer local/CI evidence convention for these cases.

3. **Brownfield comments and historical story labels add investigation noise.** Story 5.5 explicitly
   preserved older review labels in code comments because the behavior was correct. The team should
   clean misleading comments opportunistically when nearby code is already touched.

4. **Documentation lagged the final contracts.** The implementation settled the projection resource
   URI grammar and lifecycle retry wire shape, but generated project docs still carried the older
   shorthand until the post-retro documentation audit below.

## 3. Key Learnings

- **MCP is not a separate safety universe.** Agent calls must reuse the same security and identity
  invariants as generated human workflows, then add protocol-specific redaction and schema gates.
- **Opaque failure shape is a product feature.** The hidden-equivalent envelope prevents agents from
  learning whether a command exists, whether a tenant has it, or whether a schema mismatch would have
  happened.
- **Schema compatibility must be data-driven.** Story 5.5 correctly ignores caller-supplied
  compatibility booleans and derives additive/warning/breaking decisions from schema snapshots.
- **Resource URI grammar is a contract, not an example.** The generated projection resource grammar
  needs to be used consistently in docs, tests, and agent guidance.
- **Review automation needs to check evidence hygiene.** File List drift is now a repeated process
  defect across Epics 3, 4, and 5.

4. **Update agent-facing docs when Epic 6 introduces new customization guidance.**
