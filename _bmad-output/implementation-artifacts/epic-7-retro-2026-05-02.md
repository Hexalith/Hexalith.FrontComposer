## What Went Well

- **Cross-story contract tables paid off again.** Each story declared its producers/consumers explicitly, so 7-3 could consume 7-1's principal seam and 7-2's tenant snapshot without re-implementing either. The L02 lesson keeps proving itself.
- **Fail-closed-by-default discipline became muscle memory.** The persistent feedback rule on tenant/user fail-close was applied at every new boundary: missing `AuthenticationStateProvider` falls through to `NullAuthenticationStateProvider`; missing tenant snapshot blocks dispatch; missing policy gate decorates the entire `ICommandService` registration; synthetic-tenant production guard catches "default"/"anonymous" leaks.
- **Single canonical seam survived scope pressure.** `IUserContextAccessor` shape unchanged across the epic. One tenant-context accessor. One authorization evaluator. One policy registry lookup. One direct-dispatch decorator. Resisting parallel paths kept the surface coherent.
- **Architecture-fact-checking saved scope.** DN-7-3-4-1 closure proved that v1 home/capability surfaces enumerate projections only, not commands — AC13 was structurally satisfied without writing the gate. That is a counter-pattern worth remembering: investigate before implementing.
- **Counter sample stayed runnable without external credentials.** Demo path preserved via `DemoUserContextAccessor` + `AllowDemoTenantContext` + production guard. The "no-credential local dev" promise from earlier epics survived adding real auth.
- **Snapshot revalidation discipline closed real TOCTOU windows.** 7-2 P5/P6 caught the "tenant switches between resolve and `cache.SetAsync`" race; 7-3 Pass-4 P-CR4 caught the "principal disposed between `EvaluateAsync` and `StateHasChanged`" race. Without explicit revalidation oracles, these would have shipped.
- **Diagnostic governance held.** HFC2011-HFC2019, HFC1056-HFC1057, plus `AnalyzerReleases.Unshipped.md` updates, teaching messages with Got/Expected/Fix/Docs shape. Epic 6's commitment to govern diagnostic IDs as product surface carried forward without reminders.

## Challenges

- **Story 7-3 took five review passes.** Pass 1 surfaced 5 DN + 25 patches; Pass 2 confirmed all of Pass 1; Pass 3 found 6 new DN; Pass 4 found that DN4 was marked `[x]` but never implemented and that two new code paths shipped without any of Pass 3's hardening discipline; Pass 5 cleaned up 11 final structural issues. Each pass cost a full unsupervised dev session.
- **Hardening-matrix duplication was a recurring footgun.** `FcAuthorizedCommandRegion` and the renderer-emitted `RefreshPresentationAuthorizationAsync` both shipped without IDisposable, CTS capture, `_disposed` short-circuit, sequence guard, AuthenticationStateChanged subscribe, try/catch around evaluator, sync `StateHasChanged` ordering, or single-correlation-ID per refresh — the exact 12-patch hardening matrix that the form emitter received in Pass 3 had to be re-applied to two more code paths in Pass 4. Each new auth-aware component is a new copy of the same hardening discipline.
- **"Claimed `[x]` versus actually implemented" drift.** Sprint-status entries in Pass-2 / Pass-3 narrated work that the diff did not contain (DN4, DN3 partial, P-E29, sprint-status framing for Pass 4). Pass 4's Acceptance Auditor caught it. The lesson is harsh: status updates can lie in good faith when a session runs long.
- **Optional-parameter fail-OPEN regressions.** Pass-4 introduced `ICommandDispatchAuthorizationGate? authorizationGate = null` ctor params on `StubCommandService` and `EventStoreCommandClient`, then short-circuited authorization when the gate was null. That defeated D4 fail-closed. Refactor to `AuthorizingCommandServiceDecorator` removed the optional surface entirely. Optional security parameters are an anti-pattern.
- **Resource-shape divergence reappeared after DN5 was supposed to eliminate it.** `FcAuthorizedCommandRegion` consumer (`FcProjectionEmptyPlaceholder.razor`) didn't pass `BoundedContext`, so the evaluator got `BoundedContext=null` for the CTA path while form/palette/dispatch supplied it. The exact bug DN5 was created to fix. Required extending `EmptyStateCta` with `BoundedContext` and threading through the canonical registry tuple lookup. Symmetry is fragile when surfaces are added without an end-to-end resource contract test.
- **Architecture drift on tenant identity casing.** The older architecture note said tenant/user IDs should be lowercased for storage keys. Story 7-2 had to explicitly supersede that with ordinal/case-sensitive equality and document the drift in the Party-Mode Hardening Addendum. Non-trivial to discover; persistent feedback memory was the trigger to look.
- **Architecture-vs-spec contradictions surfaced under elicitation.** 7-1 DN1 (`AuthenticationStateUserContextAccessor` with sync-over-async deadlock risk shipped unregistered) was a textbook case: a half-built file with no DI registration and a `.GetAwaiter().GetResult()` deadlock. Removed entirely; WASM accessor deferred to a later story. The rule: half-built code is worse than no code.
- **Test fixture rebaselines became heavy.** 7-3 Pass 4 + Pass 5 rebaselined 9+ `verified.txt` files. Without the snapshot-rebaseline-tied-to-explicit-behavior discipline established in Epic 6, tracking which rebaseline corresponded to which patch would have been impossible.

## Key Lessons

1. **Each new auth-aware async component is a hardening-matrix copy.** When introducing components that subscribe to `AuthenticationStateChanged`, await an evaluator, and call `StateHasChanged`, the hardening matrix is fixed: IDisposable + CTS capture + `_disposed` short-circuit + sequence guard + try/catch + sync StateHasChanged + single correlation ID + AuthState subscribe/unsubscribe. Treat it as a checklist enforced by review or by extracting a base component.
2. **Optional security parameters are an anti-pattern.** `Gate? gate = null` with `if (gate is not null)` short-circuit defeats fail-closed by construction. Decorator/required-injection patterns make fail-OPEN structurally impossible.
3. **Status-narrative drift is a real review channel.** Sprint-status free-text claims are not enforced by the build. The Acceptance Auditor layer catches "claimed vs implemented" deltas; without it, the drift reaches main. AA is now load-bearing for multi-pass stories.
4. **Architecture-fact verification saves scope.** Before implementing a gate, check whether the surface enumerates the gated artifact at all. DN-7-3-4-1 saved an entire DN's worth of work by reading the code.
5. **Snapshot revalidation oracles are non-optional for delayed effects.** Cache writes after HTTP 200, Blazor `StateHasChanged` after evaluator await, SignalR rejoin after reconnect — every delayed effect has a TOCTOU window. Revalidate-or-abort on the snapshot, not on a fresh accessor read.
6. **One canonical seam beats per-surface duplication every time.** `IFrontComposerCommandPolicyRegistry.TryGetCommandPolicy(out policy, out boundedContext)` as a tuple lookup eliminated the BC-drift between `TryResolvePolicy` fast path and `ResolveBoundedContext` fallback. Two-out-parameter signatures are not pretty, but they are correct.
7. **Default-implemented interface methods preserve adopter compatibility.** Adopters with custom `IFrontComposerRegistry` automatically inherit the manifest-walk fallback for `TryGetCommandPolicy` without overriding. New canonical contracts should ship with default implementations whenever the seam allows.
8. **Half-built code is worse than no code.** `AuthenticationStateUserContextAccessor` shipping with a sync-over-async deadlock and no DI registration was a worse outcome than deferring WASM support to a later story. Delete-and-defer is a valid resolution.
9. **Localization parity is testable, not aspirational.** Pass-3 added EN/FR placeholder count parity tests across new resource keys. Catches `{0}` vs `{0}/{1}` drift at build time. Cheap discipline; should generalize.
10. **Persistent feedback memory was load-bearing.** The `feedback_tenant_isolation_fail_closed.md` rule was the trigger to question architecture's lowercase tenant rule, to refuse default constructor params for security gates, and to demand fail-closed default `AuthenticationStateProvider`. Memory is review armor.

## Significant Discoveries

Epic 7 does not invalidate Epic 8's plan, but it adds three load-bearing contracts that Epic 8 stories must consume rather than re-implement:

1. `IFrontComposerCommandPolicyRegistry.TryGetCommandPolicy(string commandFqn, out string policyName, out string? boundedContext)` is the canonical merged-manifest lookup. The default-implemented manifest-walk fallback means adopter custom registries inherit the contract for free.

2. `ITenantScopedManifestGate` is the canonical contract for tenant-scoped enumeration. Story 8-2's hallucination-rejection tool list must consume it so missing tenant context produces an empty tool list (not a synthetic default).

3. `AuthorizingCommandServiceDecorator` is the single canonical authorization site for all `ICommandService.DispatchAsync` calls. Story 8-3's two-call lifecycle dispatches commands through `ICommandService` and therefore inherits authorization enforcement automatically. MCP must not introduce a parallel dispatcher.

Epic update required: No Epic 8 rewrite. Add explicit consumption notes to Story 8-2 and Story 8-3 referencing the three contracts above. Add a Story 10-1 dependency note acknowledging the deferred Generated/Authorization bUnit suite is the natural home for cross-surface MCP coverage.
