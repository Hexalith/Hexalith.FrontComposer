**ADR-039 one-liner:** `DensityPrecedence.Resolve(userPref, deploymentDefault, surface, tier) : DensityLevel` is a pure static function implementing the four-tier UX spec §197-204 rule. Tier-force override at `tier <= Tablet → Comfortable` sits at precedence position 1 (ADR-040). Action producers (effects + components) call `Resolve` BEFORE dispatch and carry the computed `NewEffective` in the action payload — reducers stay pure static. Rejected: inline precedence per component (O(N²) maintenance), resolver inside reducer (violates Fluxor purity), lazy selector (compute per render multiplies cost by component count), `IOptions<DensityPrecedenceOptions>` (no adopter use case).

**ADR-040 one-liner:** `ViewportTier <= Tablet` forces `DensityLevel.Comfortable` regardless of `UserPreference` or `DefaultDensity` — 44 px touch target accessibility floor, not a default. `UserPreference` state is PRESERVED (not cleared) so viewport expansion re-applies the user's choice automatically. Settings dialog shows `UserPreference ?? EffectiveDensity` in the radio + an inline `FluentMessageBar Info` note when forcing is active. Rejected: responsive CSS padding per component (creates per-component inconsistency + UX spec §62 explicitly rules it out), per-surface forcing (partial fix + multiplies test matrix), force Roomy at Tablet (overshoots — UX spec §193 forbids Roomy as a factory default), clear UserPreference (destructive).

**ADR-041 one-liner (reworded post-Winston review):** `<body data-fc-density="compact|comfortable|roomy">` is the DEFAULT density source; nested `data-fc-density` on any wrapper element is a first-class override. Six scoped CSS rules in `FrontComposerShell.razor.css` — three bound to `body[data-fc-density=X]` (default) and three mirror rules bound to `[data-fc-density=X]` (local override) — set `--fc-spacing-unit: 2|4|6 px`. Every scoped CSS in the framework consumes `padding: calc(var(--fc-spacing-unit) * N)` — no per-component density logic. Current override consumer: `FcDensityPreviewPanel`. Anticipated consumers: Story 4-1 DataGrid virtualization, Story 4-5 DetailView expanded row, Story 6-5 DevModeOverlay. Rejected: per-component Fluxor subscription (N × re-render chains vs free CSS cascade), Razor `CascadingValue<DensityLevel>` (doesn't reach source-generated scoped CSS), inline body style (fights Fluent UI), `@media` queries (viewport-only; can't express user preference), per-component `data-density` on every root (unconditional — defeats zero-logic goal), "single source only, no overrides" (pretends the invariant is something the codebase already contradicts via `FcDensityPreviewPanel`). The invariant: density is expressed via `data-fc-density` + CSS cascade, never via Razor state subscriptions or inline styles.
