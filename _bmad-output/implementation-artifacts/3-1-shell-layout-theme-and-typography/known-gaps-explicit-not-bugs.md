# Known Gaps (Explicit, Not Bugs)

Each row cites a specific owning story per L10 discipline.

| ID | Gap | Owning story | Rationale for deferral |
|---|---|---|---|
| G1 | `FcShellOptions` now has 14 properties spanning 3 cross-concerns (lifecycle, forms, theming). Split into `FcLifecycleOptions` + `FcFormOptions` + `FcThemingOptions` + `FcStorageOptions` is required to prevent further accretion. | **Story 9-2** (CLI inspection & migration tools — the class split is documented as migration tooling scope because the break affects adopter configuration and warrants an automated migration) | Mid-Epic refactor is strictly riskier than continued accretion; 2-5 deliberately deferred (dev-notes "stays below the trigger; split remains Story 9-2"). 3-1 crosses the trigger but maintains the deferral. |
| G2 | Story 2-4 lifecycle copy may still be inline rather than `IStringLocalizer<FcShellResources>`-routed. 3-1 materialises the FR resource keys to unblock a follow-up PR; 3-1 does NOT touch `FcLifecycleWrapper`. | **Story 2-4 follow-up PR** (non-blocking; resource keys available after 3-1 merges) | Changing `FcLifecycleWrapper` in 3-1 cascades into Story 2-4's snapshot tests + invalidates the D17 no-auto-dismiss invariant lock. Keep the boundary clean. |
| G3 | `HFC2105` (storage skipped) stays at `Information` severity. Adopters who WANT a louder signal for fail-closed persistence skips can't escalate today; `[SuppressHFC2105]` analyzer escape hatch is also not shipped. | **Story 9-4** (Diagnostic ID system & deprecation policy) | Severity-configurable + suppression attribute require the broader diagnostic governance machinery that 9-4 is scoped to build. Shipping one-off governance in 3-1 duplicates 9-4. |
| G4 | `prefers-reduced-motion` CSS rules that would replace `FrontComposerShell`'s (nonexistent) animations with instant transitions. Since 3-1 doesn't ship any animations, this gap is technically empty — recorded for Stories 3-4 (command palette fade-in) + 4-2 (badge saturation). | **Story 10-2** (accessibility CI gates) | No animations to gate in 3-1. Gap retained for cross-story traceability. |
| G5 | `LocalStorageService.GetKeysAsync(prefix)` may fall back to per-index iteration if Task 0.4 spike reveals that `InvokeAsync<string[]>("eval", "Object.keys(localStorage)")` is blocked by the adopter's CSP. In that fallback mode, each call becomes N JS-interop round-trips — observable under load. | **Story 5-2** (ETag caching performance optimisation; that story has budget for JS-interop coalescence + potential IJSUnmarshalledRuntime use on WASM) | CSP-lockdown deployments are rare in v0.1; architecture §1144 recommends DAPR sidecar handling CORS anyway. If G5 fires in production, escalate. |
| G6 | Cross-device preference sync (user works on laptop + desktop, preferences independent on each) remains out of scope. Adopters requiring sync implement server-side storage at the application layer. | **v2** (architecture §597; UX spec §106) | Requires server-side `IStorageService` impl + per-user auth scope — too much new surface for v1. |
| G7 | Per-tenant accent / bounded-context sub-branding (different accent per bounded context in the same composed app). Framework enforces a single accent in v0.1. | **v1.x** (architecture §598) | Interacts with zero-override, customization gradient, and consistency-is-trust in ways that need empirical demand before commit. |
| G8 | RTL (`dir="rtl"`) verification — framework inherits Fluent UI's RTL behaviour but 3-1 does not ship RTL specimen baselines. | **v2** (UX spec §36) | v1 primary audience is EN + FR (LTR); adding RTL CI gates blocks shipping. |
| G9 | Windows High Contrast (`forced-colors`) mode verification + specimen baselines. Framework inherits Fluent UI behaviour but does not gate it in CI. | **Story 10-2** (accessibility CI gates) | Requires Playwright `forced-colors` emulation — Story 10-2's scope. |
| G10 | Type specimen view (DataGrid + form + nav sample + all densities) + CI-enforced screenshot diffing against committed baselines. | **Story 10-2** | Specimen infrastructure is a 2-week build; 3-1 ships the primitives (colors, typography, shell) that 10-2 assembles into the specimen. |
| G11 | `IShortcutService` + `Ctrl+K` / `Ctrl+,` / `g h` shortcut registration + conflict-warning analyzer (`HFC10xx` range). Header placeholder buttons in 3-1 don't respond to keyboard shortcuts. | **Story 3-4** (command palette & keyboard shortcuts) | Shortcut service is the natural part of the command palette foundation. 3-1 placeholder buttons are `aria-disabled`, so users don't expect keyboard interaction. |
| G12 | Responsive breakpoint behaviour (compact-desktop auto-collapse sidebar, tablet drawer nav, phone single-column) is not implemented. Shell renders at desktop dimensions on all viewports. | **Story 3-2** (sidebar navigation & responsive behaviour) | Breakpoints are intimately tied to sidebar composition; 3-2 owns the whole block. 3-1 shell ships the skeleton that 3-2 tunes. |
| G13 | Settings dialog (density radio, theme selector, live preview, Ctrl+, shortcut). The header settings icon is a placeholder that logs and does nothing. | **Story 3-3** (display density & user settings) | UI design + interactions owned by 3-3. |
| G14 | Home directory (`FcHomeDirectory`) and badge counts (`IBadgeCountService`). 3-1 shell renders whatever `ChildContent` is — it does not opinion on the home page. | **Story 3-5** (home directory, badge counts & new capability discovery) | 3-1 is layout + chrome only. |
| G15 | DataGrid state restoration (scroll, filters, sort, expanded rows). `LocalStorageService` is shipped; 3-6 consumes. | **Story 3-6** (session persistence & context restoration) | 3-1 ships the primitive; 3-6 consumes and keys properly. |
| G16 | Storage drain worker timeout (200 ms per D17) is not configurable. Adopters with specific unload-budget needs cannot tune. | **Story 9-2** (CLI inspection — typically the driver for configuration surface expansion) | Hard-coded 200 ms matches browser unload grace consistently across Chrome/Edge/Firefox. Configurability not justified at v0.1. |
