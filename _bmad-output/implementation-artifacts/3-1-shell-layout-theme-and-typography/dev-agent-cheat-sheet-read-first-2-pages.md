# Dev Agent Cheat Sheet (Read First — 2 pages)

> Amelia-facing terse summary. Authoritative spec is the full document. Every line links to a section for detail.

**Goal:** Stand up Epic 3's foundational shell chrome + theme + typography + durable storage behind the zero-override invariant. Deliver (1) `FrontComposerShell.razor` composing `FluentLayout` into Header (pinned 48 px) / Navigation (~220 px reserved, populated by Story 3-2) / Content; (2) Light/Dark/System theme toggle via Fluent UI v5 `IThemeService` with Fluxor `ThemeState` as the tenant-scoped cache; (3) configurable accent color via `FcShellOptions.AccentColor` (hex regex validated, default `#0097A7`); (4) six semantic color-slot CSS custom properties with lifecycle-state mapping spec-locked; (5) `Hexalith.FrontComposer.Typography` static class exposing 9 `Typography` enum constants (AppTitle → Title1, BoundedContextHeading → Subtitle1, ViewTitle → Title3, SectionHeading → Subtitle2, FieldLabel → Body1Strong, Body → Body1, Secondary → Body2, Caption → Caption1, Code → Body1 + monospace); (6) `LocalStorageService` WASM-compatible `IStorageService` impl with LRU eviction + fire-and-forget `SetAsync` + `FlushAsync` drain on `beforeunload`; (7) EN + FR `FcShellResources.resx` via `IStringLocalizer` + `AddHexalithShellLocalization()`; (8) refactor `ThemeEffects` + `DensityEffects` to use `IUserContextAccessor` + fail-closed on null user (per memory feedback + L03). Counter.Web rewires `MainLayout` to `<FrontComposerShell>` — no other sample changes.

**Scope boundary:** 3-1 is the **Epic 3 opener**. Sidebar collapse + responsive behaviour → **Story 3-2**. Density settings dialog + live preview → **Story 3-3**. Full command palette (FluentSearch + debounce + fuzzy match + shortcut help) → **Story 3-4**. Home directory + badge counts + "New" indicators → **Story 3-5**. DataGrid-state session resumption → **Story 3-6**. Type specimen view + screenshot diffing + axe-core CI → **Story 10-2**. Cross-device preference sync → **v2**. Per-tenant accent/branding → **v1.x** (architecture §598). 3-1's Header **HIDES** the Ctrl+K and Settings placeholder buttons entirely (compile-away `@if (false)` guard per D26 + ADR-032 — NOT rendered as aria-disabled placeholders). Story 3-3 unhides Settings when the dialog ships; Story 3-4 unhides Ctrl+K when the palette ships. The shell's parameter surface (D4) remains stable across those activations.

**Binding contracts with existing Epic 1 + 2 deliverables:**

- **Story 1-3 `FrontComposerThemeState` + `ThemeActions` + `ThemeReducers` are the authoritative Fluxor theme model** — 3-1 does NOT introduce a parallel model. `ThemeChangedAction(CorrelationId, NewTheme)` is the single write path (D7). `ThemeEffects.HandleThemeChanged` stays the persistence writer; 3-1 swaps the storage-key builder to use `IUserContextAccessor` but does NOT introduce a new effect.
- **Story 1-3 `FrontComposerDensityState` + effects** — 3-1 applies the same `IUserContextAccessor` refactor symmetrically so Density and Theme persistence share one scope-resolution pathway. Downstream Stories 3-3 (density UI) and 3-6 (session) will also call this pathway.
- **Story 2-2 `FcShellOptions.FullPageFormMaxWidth`** — 3-1 does NOT add a parallel shell-level max-width; the generated FullPage renderer already consumes this option (Story 2-2 Decision D26). `FrontComposerShell`'s `LayoutArea.Content` is unconstrained; the form itself self-limits. Test Task 10.3 proves the two agree.
- **Story 2-2 D31 `NullUserContextAccessor` fail-closed default + Story 2-2 Task 3.5 dev-mode warning** — 3-1's `ThemeEffects` + `DensityEffects` refactor reuses `NullUserContextAccessor`'s existing null-return semantics. Null tenant or user → skip persistence, log `HFC2105` Information (new), state is in-memory only until a real accessor is registered. Matches the precedent from `LastUsedValueProvider` (D31).
- **Story 2-2 `StorageKeys.BuildKey(tenantId, userId, feature[, discriminator])` helper** — 3-1 KEEPS the helper (still the right signature) but DELETES `DefaultTenantId = "default"` / `DefaultUserId = "anonymous"` constants. Every caller now supplies non-null values resolved from `IUserContextAccessor` or aborts (D8). `StorageKeys` gains no new methods.
- **Story 2-4 `FcShellOptions` append-only convention** — 3-1 adds 4 new properties (`AccentColor`, `LocalStorageMaxEntries`, `DefaultCulture`, `SupportedCultures`) → property count goes 10 → 14. The 2-4 / 2-5 dev-notes flagged splitting at ≥ 12 properties OR a 3rd cross-concern — 3-1 crosses BOTH thresholds (14 props + theme/locale is a 3rd cross-concern beyond lifecycle + forms). **Split is deferred to Story 9-2 (L10 specific story citation)** — 3-1 documents the trigger and defers to 9-2 explicitly. Adding-without-splitting matches the Story 2-5 precedent (dev-notes "stays below the trigger; split remains Story 9-2").
- **Story 2-5 `Options/FcShellOptionsThresholdValidator`** — 3-1 does NOT add cross-property validators; accent/locale/storage-entries have single-property `[RegularExpression]` / `[Range]` annotations. Validator stays at 3 lifecycle-ordering invariants.
- **Story 2-3 / 2-4 `CommandLifecycleState` color mapping** — 3-1 ships the CSS slot declarations (`--fc-color-*`) that the generated renderer + wrapper already consume indirectly via Fluent UI tokens. The slot → lifecycle-state mapping table moves from the UX spec into a single regression-locked location (`Docs/color-slots.md` committed + a `SlotMappingRegressionTests.cs` snapshot) so any future renaming surfaces in CI (D13).

**ADR-027 one-liner:** `FrontComposerShell` is a Razor Class Library component (Shell) that owns `FluentLayout` + `FluentProviders` + `Fluxor.Blazor.Web.StoreInitializer` placement. Adopter `MainLayout.razor` collapses to `<FrontComposerShell>@Body</FrontComposerShell>` — three lines. Rejected: adopter keeps their own FluentLayout and references Fc fragments; rejected because it re-exposes header composition to every adopter and defeats the zero-override strategy.

**ADR-028 one-liner:** `IThemeService.SetThemeAsync(ThemeSettings)` is the APPLIER. Fluxor `ThemeState` + `IStorageService` is the tenant-scoped CACHE that survives Blazor Server prerender → WASM handoff. Rejected: Fluent UI's built-in localStorage key → rejected because it is global (no tenant/user scoping, violates L03). We disable Fluent UI's auto-persist by never calling the overloads that enable it, and we call `SetThemeAsync(ThemeSettings)` explicitly on every hydration + change.

**ADR-029 one-liner:** `ThemeEffects` + `DensityEffects` read `TenantId` + `UserId` directly from `IUserContextAccessor` (flat `string?` properties per `IUserContextAccessor.cs:17-31` — NOT `.Current`/`User.Id`); if either fails `string.IsNullOrWhiteSpace`, the effect logs `HFC2105` and returns early — never falls back to `"default"/"anonymous"`. Rejected: continue with static defaults for "dev ergonomics" → rejected because the cost of a cross-tenant PII bleed in production greatly exceeds the cost of registering `DemoUserContextAccessor` in samples (Counter.Web already does this per Story 2-2 Task 9.4).

**ADR-030 one-liner:** `IStorageService` migrates from Singleton → Scoped. `AddHexalithFrontComposer` must `services.RemoveAll<IStorageService>()` before `services.AddScoped<IStorageService, LocalStorageService>()` — `TryAddScoped` is a no-op trap because the pre-existing Singleton descriptor wins. Rejected: `TryAddScoped` without removal (silent no-op); dual registration (config-flag indirection); keeping Singleton with `IServiceScopeFactory` lazy resolution (pushes DI complexity into infrastructure).

**ADR-031 one-liner:** Six `--fc-color-*` CSS custom properties (accent/neutral/success/warning/danger/info) declared in ONE place (`FrontComposerShell.razor.css :host`) and consumed by every downstream component via `var(--fc-color-*)`.
