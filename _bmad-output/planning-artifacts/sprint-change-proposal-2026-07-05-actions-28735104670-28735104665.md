# Sprint Change Proposal - Actions 28735104670 and 28735104665 Recovery

## 1. Issue Summary

The 2026-07-05 push to `main` at `3b96613811791b86e815d1ecf5eb318900a79acc`
failed CI and Release validation.

Evidence:

- CI `build-and-test` failed at Gate 2d Docs Validation because producer fingerprint validation was
  sensitive to CRLF-normalized checkout content for:
  - `docs/diagnostics/samples/registry-drift-report.json`

## 4. Detailed Change Proposals

### Docs Producer Fingerprints

File: `eng/validate-docs.ps1`

OLD:

```powershell
(Get-FileHash -LiteralPath $Path -Algorithm SHA256).Hash.ToLowerInvariant()
```

NEW:

```powershell
$text = Get-Content -LiteralPath $Path -Raw
$normalized = $text -replace "`r`n", "`n"
$normalized = $normalized -replace "`r", "`n"
$bytes = [System.Text.Encoding]::UTF8.GetBytes($normalized)
$hash = [System.Security.Cryptography.SHA256]::HashData($bytes)
[System.Convert]::ToHexString($hash).ToLowerInvariant()
```

Justification: producer fingerprints cover text docs/JSON artifacts and must not depend on checkout
line endings.

### Producer Fingerprint Baseline

File: `docs/validation/producer-fingerprints.json`

OLD: raw worktree hashes.

NEW: LF-normalized text hashes for all five producer inputs.

Justification: aligns the checked-in baseline with the normalized hash contract.

### Testing Package Dependency

File: `src/Hexalith.FrontComposer.Testing/Hexalith.FrontComposer.Testing.csproj`

OLD:

```xml
<PackageReference Include="Microsoft.FluentUI.AspNetCore.Components" PrivateAssets="all" />
```

NEW:

```xml
<PackageReference Include="Microsoft.FluentUI.AspNetCore.Components" />
```

Justification: `FrontComposerTestHostBuilder` directly uses Fluent UI services, so package consumers
need the same Fluent UI v5 dependency in the packed Testing package.

### Package Boundary Guard

File: `tests/Hexalith.FrontComposer.Testing.Tests/PackageBoundaryTests.cs`

OLD: the Testing package nuspec only asserted `Contracts`, `Shell`, and `bunit` dependencies.

NEW: the nuspec also asserts `Microsoft.FluentUI.AspNetCore.Components`.

Justification: prevents regression to the clean-consumer Fluent v4 fallback.
