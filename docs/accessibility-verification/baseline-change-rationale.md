# Visual Baseline Change Rationale

Story 10.2 intentionally added the first committed FrontComposer specimen visual baselines for the Chromium Windows lane. Before this update, the six `frontcomposer-type-*-chromium-win32.png` files under `tests/e2e/specs/specimen-accessibility.spec.ts-snapshots/` captured the type specimen at `1296x2510`. After the current shell/specimen expansion, the accessibility-visual lane for run `28734213263` produced the same Light/Dark by Compact/Comfortable/Roomy matrix at `1296x2812` with the lower specimen content included. Reviewers should compare the before snapshots against the matching Playwright `*-actual.png` artifacts from that run and confirm the added height reflects expected rendered content rather than clipping, overlap, or hidden accessibility controls.
