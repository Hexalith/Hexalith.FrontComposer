## Problem

HFC4001 Mcp Compatible Additive Drift Deprecated.

## Common Causes

The framework detected a condition represented by `HFC4001` while validating source metadata, runtime state, migration output, or compatibility evidence.

## How To Fix

Use the diagnostic payload fields and registry metadata to locate the affected package, API surface, and owner story. Keep values bounded, project-relative, and sanitized when reporting follow-up evidence.

## Example

```text
HFC4001: What: HFC4001 Mcp Compatible Additive Drift Deprecated. Expected: Follow the FrontComposer diagnostic contract. Fix: See https://hexalith.github.io/FrontComposer/diagnostics/HFC4001.
```

## Suppression Guidance

Suppression is `allowed-with-rationale`. Prefer `.editorconfig` or a narrowly scoped pragma for analyzer diagnostics.

## Migration/Deprecation

MCP backwards-compatible additive drift was deprecated in v0.2.0; removed in v0.4.0. See HFC4001 for the deprecation contract.

Migration steps:
1. Switch additive-drift behaviour to the explicit MCP schema-negotiation runtime gate (Story 8-6a).
2. Update host configuration to advertise the negotiated schema version rather than relying on additive tolerance.
3. Rebuild against ≥ v0.2.0 and confirm no HFC4001 diagnostics remain in MCP traffic.
