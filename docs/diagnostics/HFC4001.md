## Problem

HFC4001 Mcp Compatible Additive Drift Deprecated.

## Common Causes

The Mcp producer emits `HFC4001` when it detects `HFC4001 Mcp Compatible Additive Drift Deprecated` during FrontComposer validation or runtime reporting. Inspect the diagnostic payload for the affected symbol, generated path, package, or migration record before changing code.

## How To Fix

Follow the registry-owned message template for `HFC4001` and update the affected Mcp surface. Keep evidence project-relative and sanitized, then rerun docs validation so the help link, lifecycle state, and related diagnostics stay synchronized.

## Example

```text
HFC4001: What: HFC4001 Mcp Compatible Additive Drift Deprecated. Expected: The Mcp contract for HFC4001 is satisfied. Got: A bounded diagnostic payload identifies the affected FrontComposer surface. Fix: Apply the Mcp remediation and rerun validation.
```

## Suppression Guidance

Suppression is `allowed-with-rationale`. Prefer `.editorconfig` or a narrowly scoped pragma for analyzer diagnostics.

## Migration/Deprecation

MCP backwards-compatible additive drift was deprecated in v0.2.0; removed in v0.4.0. See HFC4001 for the deprecation contract.

Migration steps:
1. Switch additive-drift behaviour to the explicit MCP schema-negotiation runtime gate (Story 8-6a).
2. Update host configuration to advertise the negotiated schema version rather than relying on additive tolerance.
3. Rebuild against ≥ v0.2.0 and confirm no HFC4001 diagnostics remain in MCP traffic.
