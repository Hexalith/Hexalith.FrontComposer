## Problem

Custom override color styling has no forced-colors fallback.

## Common Causes

The SourceTools accessibility analyzer emits `HFC1055` at build time when statically inspectable custom override source contains custom color, border, or fill styling without `forced-colors` fallback evidence.

## How To Fix

Add a `forced-colors` fallback or remove the custom color styling from the override. Keep the fallback evidence in source inspected by the analyzer.

## Example

```text
HFC1055: What: A customization component introduces custom color styling without forced-colors evidence. Expected: Custom override colors preserve high-contrast/forced-colors usability. Got: color, border, or fill styling without forced-colors fallback. Fix: add a forced-colors fallback or use generated styling.
```

## Suppression Guidance

Suppression is `allowed-with-rationale`. Prefer `.editorconfig` or a narrowly scoped pragma for analyzer diagnostics.

## Migration/Deprecation

No migration action is required unless the registry later changes this diagnostic lifecycle.

None currently.
