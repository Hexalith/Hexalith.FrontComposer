## Problem

Duplicate Level 4 view override.

## Common Causes

The current Level 4 path reports `HFC1044` from `ProjectionViewOverrideRegistry` during registry construction when different components target the same projection and role. This is a deterministic startup hard fail. Same-component, same-version re-registration is idempotent.

## How To Fix

Keep one Level 4 replacement per projection and role, or split the replacements by distinct `ProjectionRole` values. If duplicate registration is intentional, make the repeated registration idempotent by using the same component and contract version.

## Example

```text
HFC1044: Duplicate Level 4 view overrides registered for projection OrderProjection role <any>. Expected: one replacement component. Fix: remove one registration or make one role-specific.
```

## Suppression Guidance

Suppression is `discouraged-error`. Prefer `.editorconfig` or a narrowly scoped pragma for analyzer diagnostics, and record migration or architecture rationale for Error-level suppressions.

## Migration/Deprecation

No migration action is required unless the registry later changes this diagnostic lifecycle.
