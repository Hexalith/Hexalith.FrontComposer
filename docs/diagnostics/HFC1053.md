## Problem

Custom override is missing aria-live parity.

## Common Causes

The SourceTools accessibility analyzer emits `HFC1053` at build time when statically inspectable lifecycle/status override source contains FrontComposer status markers such as `data-fc-lifecycle` or `data-fc-status` without `aria-live` parity.

## How To Fix

Add an appropriate `aria-live` region or delegate the status output back to the generated renderer. Keep the status marker and live-region evidence in source so the analyzer can validate the override.

## Example

```text
HFC1053: What: A customization component replaces status/lifecycle output without live-region parity. Expected: Status overrides preserve aria-live announcements. Got: data-fc-status or data-fc-lifecycle evidence without aria-live. Fix: add aria-live or delegate the status rendering to FrontComposer.
```

## Suppression Guidance

Suppression is `allowed-with-rationale`. Prefer `.editorconfig` or a narrowly scoped pragma for analyzer diagnostics.

## Migration/Deprecation

No migration action is required unless the registry later changes this diagnostic lifecycle.
