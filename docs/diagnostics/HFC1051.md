## Problem

Custom override has a keyboard reachability issue.

## Common Causes

The SourceTools accessibility analyzer emits `HFC1051` at build time for statically inspectable custom override components that remove the obvious keyboard path, currently pinned for `tabindex=-1` on override markup.

## How To Fix

Remove `tabindex=-1` from clickable override elements, or provide a reachable keyboard control that performs the same action. Keep the evidence in source so the analyzer can verify the fix.

## Example

```text
HFC1051: What: A customization component removes the only obvious keyboard path from an interactive element. Expected: Clickable custom override elements stay keyboard reachable. Got: tabindex=-1. Fix: Remove tabindex=-1 or provide a reachable keyboard control.
```

## Suppression Guidance

Suppression is `allowed-with-rationale`. Prefer `.editorconfig` or a narrowly scoped pragma for analyzer diagnostics.

## Migration/Deprecation

No migration action is required unless the registry later changes this diagnostic lifecycle.

None currently.
