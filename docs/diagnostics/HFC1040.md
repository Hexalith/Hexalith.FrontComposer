## Problem

Duplicate Level 3 slot override.

## Common Causes

The current Level 3 field-slot path reports `HFC1040` from `ProjectionSlotRegistry` at startup when different descriptors target the same projection, role, and field. The ambiguous tuple resolves to no slot, so generated default rendering wins. It is not currently emitted as a SourceTools build diagnostic for compile-visible slot registrations.

## How To Fix

Keep one descriptor per projection, role, and field. Remove one registration or make one override role-specific so exact-role and role-agnostic descriptors do not conflict.

## Example

```text
HFC1040: Duplicate Level 3 slot overrides registered for projection OrderProjection role <any> field Priority. Expected: one descriptor. Fix: remove one registration or make one role-specific.
```

## Suppression Guidance

Suppression is `allowed-with-rationale`. Prefer `.editorconfig` or a narrowly scoped pragma for analyzer diagnostics.

## Migration/Deprecation

No migration action is required unless the registry later changes this diagnostic lifecycle.
