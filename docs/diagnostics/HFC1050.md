## Problem

Custom override interactive element is missing an accessible name.

## Common Causes

The SourceTools accessibility analyzer emits `HFC1050` at build time for statically inspectable Level 2 templates, Level 3 slots, or Level 4 view overrides that add an `onclick` handler without visible/static accessible-name evidence such as text content, `aria-label`, `aria-labelledby`, or `title`.

## How To Fix

Add visible text or a stable name-bearing attribute to the interactive override element. Keep the fix in the override component source so the analyzer can see it, then rebuild.

## Example

```text
HFC1050: What: A customization component renders an interactive element without an accessible name. Expected: Interactive override roots expose visible text, aria-label, aria-labelledby, or an equivalent static name. Got: onclick handler without name-bearing evidence. Fix: Add visible text or an aria-label/aria-labelledby attribute.
```

## Suppression Guidance

Suppression is `allowed-with-rationale`. Prefer `.editorconfig` or a narrowly scoped pragma for analyzer diagnostics.

## Migration/Deprecation

No migration action is required unless the registry later changes this diagnostic lifecycle.
