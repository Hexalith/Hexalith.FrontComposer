## Problem

HFC0001 Query Request Filter Deprecated.

## Common Causes

The Contracts producer emits `HFC0001` when it detects `HFC0001 Query Request Filter Deprecated` during FrontComposer validation or runtime reporting. Inspect the diagnostic payload for the affected symbol, generated path, package, or migration record before changing code.

## How To Fix

Follow the registry-owned message template for `HFC0001` and update the affected Contracts surface. Keep evidence project-relative and sanitized, then rerun docs validation so the help link, lifecycle state, and related diagnostics stay synchronized.

## Suppression Guidance

Suppression is `allowed-with-rationale`. Prefer `.editorconfig` or a narrowly scoped pragma for analyzer diagnostics.

## Migration/Deprecation

QueryRequest.Filter is replaced by ColumnFilters in v0.2.0; removed in v0.4.0. See HFC0001 for the deprecation contract and the [Obsolete] DiagnosticId / UrlFormat metadata.

Migration steps:
1. Replace `QueryRequest.Filter = ...` with `QueryRequest.ColumnFilters = ...`.
2. Update any custom serializers or projections that still read the legacy `Filter` property.
3. Rebuild against ≥ v0.2.0 and confirm no HFC0001 diagnostics remain in the build log.
