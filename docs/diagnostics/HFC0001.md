## Problem

`QueryRequest.Filter` is deprecated. New code should store filter state in `QueryRequest.ColumnFilters`, where each filter remains tied to the column it scopes.

## Common Causes

The diagnostic appears when code still initializes, serializes, or reads the legacy `Filter` property. It most often shows up in adapter code that was written before column-filter support became the stable contract.

## How To Fix

Move the filter expression to `ColumnFilters`, update serializers that materialize query requests, and keep any migration evidence project-relative. Rebuild after the replacement so obsolete metadata and diagnostic links stay synchronized.

## Example

```text
HFC0001: What: QueryRequest.Filter is deprecated. Expected: Use QueryRequest.ColumnFilters for column-scoped filter expressions. Got: Legacy QueryRequest.Filter metadata was used. Fix: Move filter state to ColumnFilters before the removal release.
```

## Suppression Guidance

Suppression is `allowed-with-rationale`. Prefer `.editorconfig` or a narrowly scoped pragma for analyzer diagnostics.

## Migration/Deprecation

QueryRequest.Filter is replaced by ColumnFilters in v0.2.0; removed in v0.4.0. See HFC0001 for the deprecation contract and the [Obsolete] DiagnosticId / UrlFormat metadata.

Migration steps:
1. Replace `QueryRequest.Filter = ...` with `QueryRequest.ColumnFilters = ...`.
2. Update any custom serializers or projections that still read the legacy `Filter` property.
3. Rebuild against ≥ v0.2.0 and confirm no HFC0001 diagnostics remain in the build log.
