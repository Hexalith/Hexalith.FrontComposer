## Problem

Invalid Level 4 view override projection type.

## Common Causes

`HFC1042` is allocated for invalid Level 4 view override projection types, but the current Level 4 registration path has no compile-visible marker/parser/emitter that reports this diagnostic at build time. Treat this page as a reserved catalog entry until a Level 4 build-validation path is implemented and pinned.

## How To Fix

If you expected a build diagnostic, verify the override registration path first. Current Level 4 validation is handled by descriptor registration and registry checks for component, duplicate, and contract-version issues. Add a narrow build-validation implementation and default-lane tests before relying on `HFC1042`.

## Example

```text
HFC1042: reserved for invalid Level 4 view override projection type. Current behavior: no proven build-time emission for Level 4 registrations.
```

## Suppression Guidance

Suppression is `discouraged-error`. Prefer `.editorconfig` or a narrowly scoped pragma for analyzer diagnostics, and record migration or architecture rationale for Error-level suppressions.

## Migration/Deprecation

No migration action is required unless a future diagnostic-catalog story activates Level 4 build-time validation.
