## Problem

Invalid Level 4 view override component.

## Common Causes

The current Level 4 path reports `HFC1043` from `ProjectionViewOverrideRegistry` at startup when a descriptor component is not compatible with `ProjectionViewContext<TProjection>`. The descriptor is ignored and lower-level rendering can continue. It is not currently emitted as a SourceTools build diagnostic for Level 4 registrations.

## How To Fix

Make the replacement a Blazor component with a public `[Parameter] ProjectionViewContext<TProjection> Context` property that matches the registered projection type. Rebuild and rerun startup/tests after changing the registration.

## Example

```text
HFC1043: Invalid Level 4 view override component for projection OrderProjection. Expected: a component with [Parameter] ProjectionViewContext<OrderProjection> Context. Fix: update the replacement component or remove the descriptor.
```

## Suppression Guidance

Suppression is `allowed-with-rationale`. Prefer `.editorconfig` or a narrowly scoped pragma for analyzer diagnostics.

## Migration/Deprecation

No migration action is required unless the registry later changes this diagnostic lifecycle.

None currently.
