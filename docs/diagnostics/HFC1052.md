## Problem

Custom override suppresses focus visibility.

## Common Causes

The SourceTools accessibility analyzer emits `HFC1052` at build time when statically inspectable custom override source suppresses visible focus styling, such as `outline: none` or `box-shadow: none`, without replacement `focus-visible` evidence.

## How To Fix

Preserve the browser focus outline or add a visible `:focus-visible` replacement. Keep the fallback evidence in the override source inspected by the analyzer.

## Example

```text
HFC1052: What: A customization component suppresses visible focus styling. Expected: Custom override CSS preserves or replaces visible keyboard focus. Got: outline or box-shadow suppression without focus-visible evidence. Fix: preserve focus styling or add a visible focus-visible replacement.
```

## Suppression Guidance

Suppression is `allowed-with-rationale`. Prefer `.editorconfig` or a narrowly scoped pragma for analyzer diagnostics.

## Migration/Deprecation

No migration action is required unless the registry later changes this diagnostic lifecycle.

None currently.
