## Problem

Invalid Level 3 slot component.

## Common Causes

The current Level 3 field-slot path reports `HFC1039` at startup/runtime from `ProjectionSlotRegistry` when a descriptor component is incompatible, and from `FcFieldSlotHost` when descriptor field-type drift is detected during render. It is not currently emitted as a SourceTools build diagnostic for compile-visible slot registrations.

## How To Fix

Use a concrete Razor component that implements `IComponent` and exposes a public `[Parameter] Context` property of type `FieldSlotContext<TProjection,TField>` with a public setter. Prefer the typed `AddSlotOverride<TProjection,TField,TComponent>(...)` overload so the selected field type and component context stay aligned.

## Example

```text
HFC1039: Invalid Level 3 slot component. Expected: Razor component with [Parameter] Context of type FieldSlotContext<OrderProjection,int>. Got: OrderCountSlot. Fix: add the matching Context parameter or register a compatible component.
```

## Suppression Guidance

Suppression is `allowed-with-rationale`. Prefer `.editorconfig` or a narrowly scoped pragma for analyzer diagnostics.

## Migration/Deprecation

No migration action is required unless the registry later changes this diagnostic lifecycle.

## Related Diagnostics

None currently.
