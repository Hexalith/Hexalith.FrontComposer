## Problem

Level 4 view override contract version is incompatible.

## Common Causes

The current Level 4 path reports `HFC1045` from `ProjectionViewOverrideRegistry` at startup. Invalid or major-incompatible contract versions are ignored and lower-level rendering can continue. Minor drift is accepted and logged at Information level. It is not currently emitted as a SourceTools build diagnostic for Level 4 registrations.

## How To Fix

Rebuild the replacement component against the installed FrontComposer package and register it with `ProjectionViewOverrideContractVersion.Current`. If the major version differs, update the override package or disable the registration until it targets the current Level 4 contract.

## Example

```text
HFC1045: Level 4 view override descriptor has incompatible contract version. Expected: installed Level 4 view override contract version. Got: an older or newer incompatible major version. Fix: rebuild the replacement against the installed framework.
```

## Suppression Guidance

Suppression is `allowed-with-rationale`. Prefer `.editorconfig` or a narrowly scoped pragma for analyzer diagnostics.

## Migration/Deprecation

No migration action is required unless the registry later changes this diagnostic lifecycle.
