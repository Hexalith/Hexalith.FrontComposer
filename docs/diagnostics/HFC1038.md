## Problem

Invalid Level 3 slot selector.

## Common Causes

The current Level 3 field-slot path reports `HFC1038` from `ProjectionSlotSelector.Parse(...)` / `AddSlotOverride(...)` at adopter call-site or startup time when a selector is not a direct projection property access. It is not currently emitted as a SourceTools build diagnostic for compile-visible slot registrations.

## How To Fix

Register one slot per direct projection property, for example `x => x.Priority`. Avoid nested members, method calls, captured values, indexers, computed expressions, and unsupported conversions. Rerun startup/tests after changing the registration.

## Example

```text
HFC1038: Invalid Level 3 slot selector 'x => (x.Priority + 1)'. Expected: direct projection property expression like x => x.Priority. Fix: select one public projection property directly and register a separate slot for each field.
```

## Suppression Guidance

Suppression is `discouraged-error`. Prefer `.editorconfig` or a narrowly scoped pragma for analyzer diagnostics, and record migration or architecture rationale for Error-level suppressions.

## Migration/Deprecation

No migration action is required unless the registry later changes this diagnostic lifecycle.
