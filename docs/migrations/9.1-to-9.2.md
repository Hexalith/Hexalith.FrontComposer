# FrontComposer 9.1 to 9.2 Migration

Use this guide when a project still registers the development overlay with the old debug-overlay API or when a shipped skill-corpus example touches customization-sensitive startup code.

## Affected Versions

The change applies to adopters moving from the 9.1 development overlay contract to the 9.2 developer-mode contract. The affected package surface is the Shell startup registration path and examples that teach local diagnostics or generated-output debugging.

## Why This Changed

The versioned CLI migration contract aligned inspect/migrate output with the developer-mode overlay name used by the rest of the framework. The old debug-overlay name made diagnostics, generated-output docs, and skill-corpus examples describe the same feature with different labels.

## Old Code

```csharp no-compile reason="Shows obsolete startup API that may not exist in the current package."
using Microsoft.Extensions.DependencyInjection;

namespace Acme.Orders;

public static class FrontComposerStartup
{
    public static IServiceCollection AddOrdersFrontComposer(this IServiceCollection services)
    {
        return services.AddFrontComposerDebugOverlay();
    }
}
```

## New Code

```csharp no-compile reason="Startup extension availability depends on the adopter host package reference."
using Microsoft.Extensions.DependencyInjection;

namespace Acme.Orders;

public static class FrontComposerStartup
{
    public static IServiceCollection AddOrdersFrontComposer(this IServiceCollection services)
    {
        return services.AddFrontComposerDevMode();
    }
}
```

## Analyzer And Code Fix

`HFCM9001` reports obsolete development overlay registration API usage. No automatic code fix is published for this migration; the CLI migration report must leave the item as a manual review when the call site also configures customization or generated-output debugging behavior.

## Skill Corpus Evidence

Story 8-5 skill-corpus examples are part of the migration trigger. Any example that still shows `AddFrontComposerDebugOverlay` must be updated with the same wording as this guide or recorded with non-breaking evidence in the docs validation manifest. The expected current term is `AddFrontComposerDevMode`.

## HFCM9001

What: obsolete development overlay registration API usage.

Expected: `AddFrontComposerDevMode`.

Got: `AddFrontComposerDebugOverlay`.

Fix: replace the obsolete API with `AddFrontComposerDevMode`, then rerun `dotnet tool run frontcomposer migrate --dry-run` and docs validation.

## HFCM9002

What: customization-sensitive migration marker.

Expected: developer review before changing behavior.

Fix: review the affected call site manually and keep at least one minor version of deprecation notice before removal.
