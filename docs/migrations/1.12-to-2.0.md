# FrontComposer 1.12 to 2.0 Migration

Use this guide when upgrading an adopter from the stable `v1.12.0` package set to `2.0.0`.
The Release Owner approved a major release because public types moved between assemblies without
type forwarding and runtime/testing-owned types moved to new namespaces.

## Affected Versions

This migration applies to `Hexalith.FrontComposer` `1.12.x` consumers moving to `2.0.0`.
It affects rendering/shortcut consumers, Shell and Testing integrations, and code using the
flattened `QueryRequest` criteria surface.

The explicit `2.0.0` package set is:

- `Hexalith.FrontComposer.Cli`
- `Hexalith.FrontComposer.Contracts`
- `Hexalith.FrontComposer.Contracts.UI`
- `Hexalith.FrontComposer.Mcp`
- `Hexalith.FrontComposer.Schema`
- `Hexalith.FrontComposer.Shell`
- `Hexalith.FrontComposer.SourceTools`
- `Hexalith.FrontComposer.Testing`

## Why This Changed

`Hexalith.FrontComposer.Contracts` is now a UI-clean `net10.0;netstandard2.0` kernel. Blazor and
Fluent contracts moved to the net10-only Contracts.UI package so SourceTools and other kernel
consumers do not inherit UI dependencies. Runtime options, registries, and Fluxor actions moved to
Shell; the in-memory storage fake moved to Testing. Query criteria are composed separately from
tenant, EventStore routing, validator, ETag, and cache metadata.

The rendering and shortcut move preserves source namespaces, but the assembly identity changed
without `TypeForwardedTo`; existing binaries must add Contracts.UI and rebuild. The Story 11.12
moves change both namespace and assembly and therefore require source edits.

## Old Code

```csharp no-compile reason="Shows the deprecated 1.12 flattened query and former type owners."
using System.Collections.Immutable;

using Hexalith.FrontComposer.Contracts;
using Hexalith.FrontComposer.Contracts.Communication;
using Hexalith.FrontComposer.Contracts.Rendering;
using Hexalith.FrontComposer.Contracts.Storage;

FcShellOptions options = new();
InMemoryStorageService storage = new();

LoadPageAction action = new(
    "orders",
    0,
    25,

    null,
    false,
    null,
    completion,
    CancellationToken.None);

QueryRequest request = new(
    ProjectionType: "OrdersProjection",
    TenantId: "tenant-a",
    Skip: 0,
    Take: 25,
    SearchQuery: "open",
    SortColumn: "CreatedAt",
    SortDescending: true);
```

## New Code

Add `Hexalith.FrontComposer.Contracts.UI` wherever source uses typography, render-fragment
customization contexts, or keyboard-event shortcut APIs. Reference Shell or Testing for their owned
implementations.

```csharp no-compile reason="Illustrative adopter migration omits package versions and host wiring."
using System.Collections.Immutable;

using Hexalith.FrontComposer.Contracts.Communication;
using Hexalith.FrontComposer.Shell.Options;
using Hexalith.FrontComposer.Shell.State.DataGridNavigation;
using Hexalith.FrontComposer.Testing;

FcShellOptions options = new();
InMemoryStorageService storage = new();
TaskCompletionSource<object> completion = new(TaskCreationOptions.RunContinuationsAsynchronously);
LoadPageAction action = new(
    "orders",
    0,
    25,
    ImmutableDictionary<string, string>.Empty,
    null,
    false,
    null,
    completion,
    CancellationToken.None);

QueryRequest request = QueryRequest.Create(
    new ProjectionQuery(
        "OrdersProjection",
        Skip: 0,
        Take: 25,
        SearchQuery: "open",
        SortColumn: "CreatedAt",
        SortDescending: true),
    TenantId: "tenant-a");
```

Direct JSON serialization remains flat and contains no `criteria` member. The legacy `Filter`
property also remains only for compatibility with existing EventStore payload forwarding.

## Analyzer And Code Fix

HFC0001 reports use of the flattened v1.12 `QueryRequest` constructor, criteria properties,
deconstruction, or `Filter` on net10. `CS0618` is the netstandard2.0 fallback. No automatic code fix
is shipped: create a `ProjectionQuery`, pass it to `QueryRequest.Create`, and access criteria through
`request.Criteria`. The shims remain functional throughout 2.x and are targeted for removal in
`3.0.0`.

## Compatibility And Non-Changes

This migration does not incidentally change:

- `IQueryService.QueryAsync` or Testing query callback signatures;
- EventStore request bodies, headers, cache eligibility/keys, or tenant behavior;
- MCP output or generated descriptors;
- schema fingerprint algorithms/material;
- CLI JSON schemas;
- generated command routes or generated-output paths;
- Pact wire shape.

Fluent UI v5 and bUnit remain prerelease dependencies. Scoped NU5104 suppressions document the
intentional stable-on-prerelease posture; they do not guarantee risk-free dependency resolution.
